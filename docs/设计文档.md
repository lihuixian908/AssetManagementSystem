# 固定资产管理系统 - 设计文档

## 一、项目概述

### 1.1 项目背景
为公司开发一套固定资产管理系统，实现对公司固定资产的全生命周期管理，包括资产入库、领用、归还、维修、报废等业务流程，提高资产管理效率，降低资产流失风险。

### 1.2 项目目标
- 实现资产的数字化管理，替代传统手工台账
- 提供资产全生命周期追踪能力
- 实现资产审批流程电子化
- 提供数据统计分析报表
- 支持多角色权限管理
- 实现设备二维码扫描，便捷管理设备信息更新
- 增加设备变更的信息记录功能

### 1.3 技术栈
| 层级 | 技术 | 版本 |
|------|------|------|
| 前端框架 | Vue 3 | 3.x |
| 前端语言 | TypeScript | 5.x |
| UI组件库 | Element Plus | 2.x |
| 状态管理 | Pinia | 2.x |
| 路由管理 | Vue Router | 4.x |
| HTTP客户端 | Axios | 1.x |
| 构建工具 | Vite | 5.x |
| 后端框架 | FastAPI | 0.137.x |
| 后端语言 | Python | 3.14 |
| 数据库 | MySQL | 8.0 |
| ORM框架 | SQLAlchemy | 2.0 |
| 数据库迁移 | Alembic | 1.18 |
| 认证方案 | JWT (python-jose) | 3.5 |
| ASGI服务器 | Uvicorn | 0.49 |

### 3.2 数据表设计

#### 3.2.1 用户表 (users)
| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INT | PK, AUTO_INCREMENT | 主键 |
| username | VARCHAR(50) | UNIQUE, NOT NULL | 用户名 |
| password | VARCHAR(255) | NOT NULL | 密码(加密) |
| real_name | VARCHAR(50) | NOT NULL | 真实姓名 |
| department | VARCHAR(100) | | 所属部门 |
| role | VARCHAR(20) | NOT NULL | 角色(admin/user) |
| phone | VARCHAR(20) | | 手机号 |
| email | VARCHAR(100) | | 邮箱 |
| status | TINYINT | DEFAULT 1 | 状态(0禁用/1启用) |
| created_at | DATETIME | DEFAULT NOW | 创建时间 |
| updated_at | DATETIME | ON UPDATE | 更新时间 |

#### 3.2.2 资产分类表 (categories)
| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INT | PK, AUTO_INCREMENT | 主键 |
| name | VARCHAR(100) | NOT NULL | 分类名称 |
| code | VARCHAR(50) | UNIQUE, NOT NULL | 分类编码 |
| parent_id | INT | DEFAULT 0 | 父分类ID |
| sort_order | INT | DEFAULT 0 | 排序 |
| status | TINYINT | DEFAULT 1 | 状态 |
| created_at | DATETIME | DEFAULT NOW | 创建时间 |
| updated_at | DATETIME | ON UPDATE | 更新时间 |

#### 3.2.3 资产表 (assets)
| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INT | PK, AUTO_INCREMENT | 主键 |
| asset_code | VARCHAR(50) | UNIQUE, NOT NULL | 资产编号 |
| name | VARCHAR(200) | NOT NULL | 资产名称 |
| category_id | INT | FK, NOT NULL | 分类ID |
| department | VARCHAR(100) | NOT NULL | 所属部门 |
| user_id | INT | FK, NULL | 使用人 |
| location | VARCHAR(200) | | 存放位置 |
| status | VARCHAR(20) | NOT NULL | 状态 |
| price | DECIMAL(10,2) | | 购买价格 |
| purchase_date | DATE | | 购买日期 |
| description | TEXT | | 描述 |
| images | JSON | | 图片列表 |
| created_at | DATETIME | DEFAULT NOW | 创建时间 |
| updated_at | DATETIME | ON UPDATE | 更新时间 |

**资产状态枚举**: idle(闲置), in_use(使用中), borrowed(出借中), maintenance(维修中), scrapped(已报废), lost(丢失)

#### 3.2.4 资产操作记录表 (asset_records)
| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INT | PK, AUTO_INCREMENT | 主键 |
| asset_id | INT | FK, NOT NULL | 资产ID |
| user_id | INT | FK | 操作人ID |
| type | VARCHAR(20) | NOT NULL | 操作类型 |
| description | TEXT | | 操作描述 |
| operator | VARCHAR(50) | NOT NULL | 操作人 |
| created_at | DATETIME | DEFAULT NOW | 创建时间 |

**操作类型枚举**: create(入库), assign(领用), return(归还), transfer(调拨), maintenance(维修), scrap(报废), update(更新), scan(扫码), borrow(出借)

## 六、权限设计

### 6.1 角色定义
| 角色 | 说明 | 权限范围 |
|------|------|----------|
| 超级管理员(admin) | 系统管理员 | 所有权限 |
| 普通用户(user) | 普通员工 | 查看自己相关资产、申请领用 |

### 6.2 权限控制
- **路由级权限**：根据角色控制可访问的页面
- **按钮级权限**：根据角色控制操作按钮的显示
- **数据级权限**：普通用户只能查看自己相关的资产数据

## 七、安全设计

### 7.1 认证方案
- 采用 JWT (JSON Web Token) 进行身份认证
- Token 有效期：1440分钟（24小时）
- 密码采用 bcrypt 加密存储

### 7.2 安全措施
- CORS 跨域配置
- 请求频率限制
- SQL注入防护（SQLAlchemy ORM）
- XSS防护
- 敏感操作日志记录

## 八、开发规范

### 8.1 后端规范
- 遵循 RESTful API 设计规范
- 统一响应格式：
```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```
- 错误处理统一使用 HTTP 状态码
- 使用 Alembic 进行数据库版本管理

### 8.2 前端规范
- 使用 ESLint + Prettier 进行代码规范
- 组件命名采用 PascalCase
- 文件命名采用 kebab-case
- 使用 Composition API (setup语法糖)
