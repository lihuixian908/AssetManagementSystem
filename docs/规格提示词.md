# 固定资产管理系统重构需求文档

## 一、项目背景

当前项目为企业固定资产管理系统。

### 技术栈

#### 前端

- Vue3
- TypeScript
- Element Plus
- Pinia
- Vue Router
- Axios

#### 后端

- FastAPI
- SQLAlchemy
- MySQL
- Pydantic
- JWT

### 当前菜单结构

```text
首页
资产管理
用户管理
分类管理
```

### 当前已实现功能

- 资产管理
- 分类管理
- 用户管理
- 首页统计

### 当前存在的问题

- 资产管理和分类管理功能重复
- 资产管理页面承担了设备管理功能
- 分类管理页面只负责分类树管理
- 用户操作路径复杂
- 设备信息需要进入资产管理页面才能查看
- 分类存在二级结构，不符合实际业务需求

### 重构目标

将系统调整为：

```text
首页
用户管理
分类管理
```

统一由「分类管理」承担：

- 分类维护
- 设备查询
- 设备新增
- 设备编辑
- 设备删除

## 二、菜单结构调整

### 当前菜单

```text
首页
资产管理
用户管理
分类管理
```

### 修改后菜单

```text
首页
用户管理
分类管理
```

### 删除内容

- 删除资产管理菜单
- 删除资产管理页面
- 删除资产管理路由
- 删除资产管理组件
- 删除资产管理菜单配置

### 保留内容

- 保留资产数据库表
- 保留资产数据
- 保留资产相关接口
- 保留首页统计功能

### 注意事项

仅删除页面入口。

不得删除任何设备数据。

## 三、分类管理模块重构

### 重构目标

分类管理升级为：

```text
设备分类与设备管理中心
```

即：

```text
分类管理
=
分类维护
+
设备查询
+
设备新增
+
设备编辑
+
设备删除
```

以后所有设备管理统一在分类管理页面完成。

---

## 四、分类结构调整

### 当前结构

```text
电子设备
├─ 电脑
├─ 打印机

办公家具
```

### 修改后结构

```text
电子设备

办公家具

服务器设备

测试设备

网络设备

其他设备
```

### 分类规则

仅保留一级分类：

- 取消树形分类
- 取消父子级结构
- 取消二级分类
- 取消三级分类

### 前端调整

删除：

- Tree组件
- TreeSelect组件
- 树形展示逻辑

改为：

- 分类标签
- 分类列表
- 分类按钮组

### 数据库调整

当前表：

```sql
asset_category
```

包含：

```sql
parent_id
```

要求：

- 前端不再使用 parent_id
- 后端不再处理父子关系
- 分类统一为一级分类

可以保留字段兼容历史数据。

## 六、设备查询功能迁移

将原资产管理页面中的查询功能迁移至分类管理页面。

### 查询条件

- 资产编号
- 设备名称
- 设备状态
- 所属部门
- 负责人

### 查询方式

模糊查询

```text
输入 ThinkPad
返回所有 ThinkPad 设备
```

```text

```

```text

- 资产编号
- 设备名称
- 分类
- 状态
- 所属部门
- 负责人

## 九、分页功能

```

## 十一、接口调整

### 查询设备列表

```http
GET /api/assets
```

### 查询参数

| 参数 | 类型 |
|--------|--------|
| keyword | string |
| category_id | int |
| status | string |
| department_id | int |
| owner | string |
| page | int |
| page_size | int |

### 返回格式

```json
{
  "total": 356,
  "page": 1,
  "page_size": 10,
  "items": []
}
```

### 新增设备

```http
POST /api/assets
```

### 编辑设备

```http
PUT /api/assets/{id}
```

### 删除设备

```http
DELETE /api/assets/{id}
```
